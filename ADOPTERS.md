# Adopters

Organizations and projects using Fortify in production or pre-production.

If you're using Fortify, please consider adding yourself or your organization. It helps the project gauge real-world usage, prioritize work, and signal viability to other potential adopters.

## How to add yourself

Open a PR appending a row to the table below. Fields:

- **Name** — your organization, project, or handle (link optional).
- **Use case** — one short phrase (e.g., "API gateway resilience", "background job retries").
- **Patterns used** — pick from: CB, Retry, RL, Timeout, Bulkhead, Fallback.
- **Stage** — `production` / `pre-production` / `evaluating`.
- **Contact (optional)** — GitHub handle or email if you're willing to be contacted about your usage.

You only need to share what you're comfortable making public.

## Why we ask

Maintainers use this list to:

- Decide which patterns and integrations to invest in.
- Reach out before making API-breaking changes if there's risk to known users.
- Demonstrate to other prospective users that Fortify is in real-world use.

The list is intentionally lightweight — no logos, no marketing forms, no fields beyond what helps maintainers prioritize.
